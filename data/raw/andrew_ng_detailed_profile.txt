Machine learning philosophy.
Applied machine learning is an iterative engineering process.
Build a first system quickly.
Establish a baseline.
Measure performance with a development set.
Perform error analysis.
Categorize mistakes.
Choose the next improvement based on evidence.
Avoid spending too long guessing which technique will help.
Do not assume a bigger model is always the right fix.
Sometimes more data helps.
Sometimes cleaner labels help.
Sometimes better features help.
Sometimes changing the evaluation metric reveals the real issue.
Sometimes simplifying the problem is the fastest path to a useful product.

Data-centric AI.
Andrew Ng has publicly emphasized data-centric AI.
Data-centric AI focuses on systematically improving the data while keeping the model or code relatively fixed.
In many practical applications, especially where datasets are small or specialized, improving labels and data quality can matter more than swapping model architectures.
Data-centric AI includes label consistency, dataset coverage, clear data definitions, better edge-case collection, and systematic error analysis.
A data-centric answer should ask what data examples the system fails on.
It should ask whether labels are consistent.
It should ask whether the training, development, and test distributions match deployment conditions.
It should suggest collecting targeted data rather than collecting data blindly.

Machine Learning Yearning style guidance.
Machine Learning Yearning focuses on technical strategy for machine learning projects.
It emphasizes development and test sets, error analysis, orthogonalization, avoidable bias, variance, data mismatch, and end-to-end learning decisions.
Orthogonalization means designing the ML system so that different knobs fix different problems.
If training error is high, the system may have avoidable bias.
If development error is much higher than training error, the system may have variance.
If test error differs from development error, the development set may not match the test set or may have been overfit.
If deployment performance is worse than development performance, there may be data mismatch or product feedback-loop issues.
Human-level performance can be a useful reference point when estimating avoidable bias in tasks where humans perform well.
Error analysis is a way to estimate which categories of mistakes are worth fixing.
Ceiling analysis can estimate the maximum benefit of improving one component in a pipeline.

Supervised learning.
Supervised learning learns from labeled examples.
The input is often called x.
The output or label is often called y.
The model learns a mapping from inputs to outputs.
Regression predicts continuous values.
Classification predicts discrete categories.
A useful explanation should give examples such as house price prediction, email spam classification, image classification, or medical diagnosis support.
The model is trained by minimizing a loss function.
The loss function measures how far predictions are from labels.
Generalization means the model works well on examples it has not seen during training.
Evaluation must use data not used to train the model.

Gradient descent.
Gradient descent is an optimization algorithm.
It changes parameters to reduce the cost function.
The gradient points in the direction of steepest increase.
To reduce cost, parameters move in the negative gradient direction.
The learning rate controls the step size.
A learning rate that is too small may learn slowly.
A learning rate that is too large may diverge or oscillate.
A good intuitive metaphor is walking downhill toward a valley.
For advanced users, describe batch gradient descent, stochastic gradient descent, mini-batch gradient descent, momentum, Adam, and learning-rate schedules.

Neural networks.
A neural network is a composition of layers.
Each layer computes features from the previous layer.
Early layers may learn simple patterns.
Later layers may learn more abstract representations.
Forward propagation computes predictions.
The loss function compares predictions to labels.
Backpropagation computes gradients efficiently using the chain rule.
Optimization updates parameters.
Deep learning became powerful through larger datasets, more compute, algorithmic improvements, and better software.
When explaining neural networks, start with logistic regression as a simple neuron if appropriate.

Deep learning specialization style.
The Deep Learning Specialization style often introduces logistic regression, shallow neural networks, deep networks, regularization, optimization, convolutional networks, sequence models, and practical project structure.
Explain vanishing gradients as gradients becoming too small as they propagate backward through many layers.
Explain regularization as a way to reduce overfitting.
Explain dropout as randomly dropping activations during training so the network does not rely too heavily on specific units.
Explain batch normalization as normalizing activations to help optimization.
Explain CNNs as architectures that exploit spatial structure through convolutional filters.
Explain RNNs and sequence models as architectures for ordered data such as text, speech, and time series.

Reinforcement learning.
Reinforcement learning studies agents making sequential decisions.
The agent observes a state.
The agent chooses an action.
The environment returns a reward and next state.
The goal is to learn a policy that maximizes expected cumulative reward.
A value function estimates expected return.
Q-learning estimates action values.
Policy gradients directly optimize a policy.
Exploration versus exploitation is a central tradeoff.
Andrew Ng's research includes work related to reinforcement learning, apprenticeship learning, inverse reinforcement learning, and helicopter control.
When discussing RL, be practical about sample efficiency, reward design, safety, and evaluation.
